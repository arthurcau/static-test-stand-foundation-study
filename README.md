# Foundation Analysis Toolkit

> **⚠️ SAFETY WARNING**
> This project provides preliminary structural and geotechnical checks for simplified pile-group foundations under lateral and axial loading. It is intended for educational and portfolio use, not for final design or construction approval. It does not replace site-specific geotechnical investigation, certified design procedures, or review by qualified engineers.

## Overview
This repository contains a Python-based educational toolkit for preliminary structural and geotechnical evaluation of simplified pile-group foundations under lateral and axial loading. 

The project combines two modeling approaches: a Broms-type limit equilibrium estimate for preliminary lateral resistance checks and a simplified 1D beam-on-nonlinear-Winkler-foundation model using p-y soil springs. The tool reports service deflection, structural stress estimates, simplified pullout checks and design-space classifications.

The code is organized using a modular structure, separating calculation routines, data files, plotting utilities and the graphical interface. This makes the code easier to test, maintain and reuse.

## Functionality
The toolkit evaluates selected configurations and reports service deflection, structural utilization and simplified pullout safety factors. The GUI takes the following inputs:
- **Applied load (N)** and **load application height (m)**.
- **Pile diameter (m)** and **embedment length (m)**.
- **Total piles**, **number of rows**, and **row spacing (m)**.
- **Geotechnical Parameters**: Cohesive soils with specific undrained shear strength ($C_u$), unit weight ($\gamma$), and the empirical $\varepsilon_{50}$ parameter used in p-y curve estimation.
- **Material Properties**: Steel yielding limits ($S_y$).

### Engineering Outputs
The toolkit can generate stress profiles along the pile depth and design-space maps for selected geometric parameters.
- **Stress Profiles**: Estimates the axial, bending, and shear stresses along the depth of the most critical pile, overlaying the von Mises yield criterion against the material's yield strength.
- **L-D Design Map (Embedment Length vs. Pile Diameter)**: The design-space map classifies combinations of embedment length and pile diameter according to the governing failure criterion. The map is generated by running a nonlinear parameter sweep over the selected design space. Non-convergent cases are treated as failed configurations in the design-space classification.

## Methodologies and Theoretical Limitations

### 1. Analytical Limit Equilibrium (Broms, 1964)
**Methodology:**
Calculates ultimate lateral resistance using static equilibrium, assuming the soil acts as a perfectly rigid-plastic material. It estimates the ultimate capacity without computing actual deflections.

**Limitations:**
- **No Serviceability Prediction:** It does NOT estimate actual deflections (mm) prior to failure.
- **Kinematic Assumptions:** Assumes the pile behaves as either perfectly rigid (rotating around a point) or perfectly flexible (forming a plastic hinge).
- **Conservatism:** Highly conservative and generally ignores complex group effects, relying on empirical passive pressure multipliers.

### 2. Simplified 1D Beam-on-Nonlinear-Winkler-Foundation Model
**Methodology:**
Models the pile as a 1D elastic beam discretized into finite elements, supported by non-linear soil springs. The soil behavior is governed by **Matlock-Reese p-y curves**, capturing the degradation of soil stiffness as lateral deflection increases. Axial pullout capacity is estimated through a simplified side-friction model.

**Limitations:**
- **Parameter Sensitivity:** Highly sensitive to the exactness of inputted soil parameters ($\gamma$, $\varepsilon_{50}$).
- **Geotechnical Scope:** The specific p-y curve models implemented are intended primarily for soft-to-medium cohesive soils. The nonlinear solver includes basic convergence checks and reports non-convergent cases as failed configurations.

## Validation Status
The implementation has been checked for numerical consistency using simplified test cases. It has not yet been validated against field tests, laboratory data or certified commercial software.

## Known Limitations
The model assumes homogeneous cohesive soil, circular steel piles, simplified pile-group load distribution and quasi-static loading. These assumptions are used to keep the analysis transparent and reproducible. The current implementation does not account for:
- Layered soils
- Cyclic loading
- Dynamic effects
- Installation effects
- Pile-to-pile interaction beyond the simplified load distribution model.

## Example Use
You can also use the engine directly without the GUI:
```python
import sys
sys.path.append('./src')
import foundation_core as fcore

# Run Broms analysis
res = fcore.evaluate_capacity_broms(
    applied_force_n=3000, eccentricity_m=0.05, passive_zone_factor=1.5,
    number_of_piles=8, diameter_m=0.01, embedded_length_m=0.5,
    soil_cu_pa=100000, steel_sy_pa=250e6
)
print("Broms Result:", res['approved'])
```
